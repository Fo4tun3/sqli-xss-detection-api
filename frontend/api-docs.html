FAQ

Frequently asked questions about the API...

Q1: What is the purpose of this API?

A1: This API is designed to detect SQL injection (SQLi) and Cross-Site Scripting (XSS) vulnerabilities in web applications using machine learning algorithms. It helps developers secure their applications by identifying and mitigating these common security threats.



Q2: How do I get started with the API?

A2: To get started, you need to sign up for an API key, which you can obtain from our website. Once you have the API key, you can integrate the API into your web application by following the instructions provided in the "Getting Started" section.



Q3: What are the main endpoints of this API?

A3: The main endpoints of this API are:

/detect-sqli-xss: Detects SQL injection vulnerabilities.



Q4: How do I use the /detect-sqli endpoint?

A4: To use the /detect-sqli endpoint, send a POST request with the following JSON payload:


{
"method": "POST",
"endpoint": "/detect-sqli",
"body": {
    "query": "SELECT * FROM users WHERE id = 1"
}
}

The response will indicate whether the provided Payload is vulnerable to SQL injection or XSS attack.



Q6: How accurate is the API in detecting vulnerabilities?

A6: Our machine learning models are trained on a large dataset of known vulnerabilities and are continuously updated to improve accuracy. However, we recommend using this API as part of a comprehensive security strategy.



Q7: Is there a limit to the number of requests I can make?

A7: Yes, there are rate limits in place to ensure fair usage. Please refer to our pricing page for details on request limits for different subscription plans.



Q8: Can I use this API in a production environment?

A8: Yes, this API is designed for both development and production environments. However, we recommend thorough testing in a staging environment before deploying to production.



Q9: What kind of support is available if I encounter issues?

A9: We offer technical support via email and a community forum where you can ask questions and share experiences with other developers. For urgent issues, you can contact our support team directly.



Q10: How do I report a bug or suggest a feature?

A10: To report a bug or suggest a feature, please visit our GitHub repository and open an issue. We appreciate your feedback and contributions to improve the API.
