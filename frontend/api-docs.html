Getting Started

To start using the API, follow these steps...



Using JavaScript Fetch


JavaScript code

    fetch('http://127.0.0.1:5000/predict-sqli-xss', {
        method: 'POST',
        headers: {
            'Content-Type': 'application/json',
        },
        body: JSON.stringify({ query: 'your input text' }),
    })
    .then(response => response.json())
    .then(data => {
        console.log('Prediction:', data.prediction);
        console.log('Predicted Class:', data.predicted_class);
    })
    .catch((error) => {
        console.error('Error:', error);
    });


Using Python to Make a POST Request

Step-by-Step Guide

1. Install the requests Library: If you don't have the requests library installed, you can install it using pip.


bash code

    pip install requests

2. Write the Python Script: Create a Python script to send a POST request to the /predict-sqli-xss endpoint with the input text.


python code

    import requests
    import json

    # Define the API endpoint
    url = 'http://your-api-url.com/api/v1/predict-sqli-xss'

    # Define the text input
    data = {
        'query': 'your input text'
    }

    # Set the headers
    headers = {
        'Content-Type': 'application/json'
    }

    # Send the POST request
    response = requests.post(url, headers=headers, data=json.dumps(data))

    # Check if the request was successful
    if response.status_code == 200:
        # Parse the JSON response
        result = response.json()
        print('Prediction:', result['prediction'])
        print('Predicted Class:', result['predicted_class'])
    else:
        print('Error:', response.status_code, response.text)


Using Postman

1. Open Postman and create a new POST request.

2. Enter the URL: http://your-api-url.com/api/v1/predict-sqli-xss.

3. Go to the "Headers" tab and set Content-Type to application/json.

4. Go to the "Body" tab, select raw and enter the JSON data:


json code

    {
        "query": "your input text"
    }

5. Click Send to make the request.


Example Response

A successful request to the /predict endpoint will return a JSON response similar to the following:


json code

    {
        "prediction": {
            "SQL Injection": 0.8814384341239929,
            "Cross-Site Scripting (XSS)": 0.005745088681578636,
            "Normal": 0.1128164678812027
        },
        "predicted_class": "SQL Injection"
    }

This response includes the probabilities for each class and the predicted class based on the input text.

FAQ

Frequently asked questions about the API...

Q1: What is the purpose of this API?

A1: This API is designed to detect SQL injection (SQLi) and Cross-Site Scripting (XSS) vulnerabilities in web applications using machine learning algorithms. It helps developers secure their applications by identifying and mitigating these common security threats.



Q2: How do I get started with the API?

A2: To get started, you need to sign up for an API key, which you can obtain from our website. Once you have the API key, you can integrate the API into your web application by following the instructions provided in the "Getting Started" section.



Q3: What are the main endpoints of this API?

A3: The main endpoints of this API are:

/detect-sqli-xss: Detects SQL injection vulnerabilities.



Q4: How do I use the /detect-sqli endpoint?

A4: To use the /detect-sqli endpoint, send a POST request with the following JSON payload:


    {
    "method": "POST",
    "endpoint": "/api/v1/predict-sqli-xss",
    "body": {
        "query": "SELECT * FROM users WHERE id = 1"
    }
    }

The response will indicate whether the provided Payload is vulnerable to SQL injection or XSS attack.
